¿Que es Django?

Django es un framework de Python que facilita la creación de sitios web utilizando Python.Django se encarga de las cosas difíciles para que usted pueda concentrarse en crear sus aplicaciones web.Django enfatiza la reutilización de componentes, también conocida como DRY (Don't Repeat Yourself), y viene con características listas para usar como sistema de inicio de sesión, conexión de base de datos y operaciones CRUD (Crear Leer Actualizar Eliminar).


¿Como funciona Django?

Django sigue el patron de diseño MVT(Modelo,Vista,Controlador)

  • Modelo:Los datos que desea presentar, generalmente datos de una base de datos.
  • Vista:un controlador de solicitudes que devuelve la plantilla y el contenido relevantes, según la solicitud del usuario.
  • Plantilla:un archivo de texto (como un archivo HTML) que contiene el diseño de la página web, con lógica sobre cómo mostrar los datos.

¿Cual es su Estructura?

Modelo

El modelo proporciona datos de la base de datos.En Django, los datos se entregan como un Mapeo Relacional de Objetos (ORM), que es una técnica diseñada para facilitar el trabajo con bases de datos.La forma más común de extraer datos de una base de datos es SQL. Un problema con SQL es que se requiere un buen conocimiento de la estructura de la base de datos para poder trabajar con ella.Django, con ORM, facilita la comunicación con la base de datos, sin tener que escribir sentencias SQL complejas.Los modelos normalmente se encuentran en un archivo llamado models.py.

Vista

Una vista es una función o método que toma solicitudes http como argumentos, importa los modelos relevantes, descubre qué datos enviar a la plantilla y devuelve el resultado final.Las vistas normalmente se encuentran en un archivo llamado views.py.

Plantilla

Una plantilla es un archivo donde se describe cómo se debe representar el resultado.Las plantillas suelen ser archivos .html, con código HTML que describe el diseño de una página web, pero también pueden estar en otros formatos de archivo para presentar otros resultados, pero nos concentraremos en los archivos html.Las plantillas de una aplicación se encuentran en una carpeta llamada templates.

URL

Django también proporciona una forma de navegar por las diferentes páginas de un sitio web.Cuando un usuario solicita una URL, Django decide a qué vista la enviará.Esto se hace en un archivo llamado urls.py.


Ventajas

  • Rapido Desarrollo
  • Seguridad Integrada
  • Potente panel de administracion
  • Facilidad de aprendizaje

Desventajas

  • Rendimiento mas lento para ciertas aplicaciones
  • Su aprendizaje es mas complejo
  • Requiere una gran cantidad de recursos para proyectos grandes y complejos
  • Tiene una gran cantidad de depemdencias lo que lo puede dificultar su instalacion y configuracion en algunos entornos
